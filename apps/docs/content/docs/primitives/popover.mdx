# Popover

## Overview

Popover is a headless compound primitive for non-modal overlay content. It matches Radix UI's API: Root, Trigger, Anchor, Portal, Content, and Close. Non-modal by default; focus is not trapped.

**Anatomy:** Root → Trigger, Portal → Content (Close). Optional Anchor for positioning reference.

## Accessibility

### Keyboard

- **Escape** — Closes the popover
- **Enter** / **Space** — Activate Trigger and Close
- **Tab** / **Shift+Tab** — Move focus (not trapped when modal=false)

### Focus

- Focus moves to first focusable element when popover opens (when modal)
- Focus returns to trigger when popover closes
- Non-modal: focus is not trapped; user can tab outside

### Screen Reader

- Content: `role="dialog"` when modal, or appropriate role for non-modal
- `data-state` for open/closed
- Accessible name via `aria-label` or `aria-labelledby`

### Consumer Responsibilities

You must provide:

- Visible focus indicators
- Positioning (CSS, Popper, Floating UI, etc.)
- Content styling

### Controlled

```tsx
const [open, setOpen] = useState(false);

<Popover.Root open={open} onOpenChange={setOpen}>
  <Popover.Trigger>Open</Popover.Trigger>
  <Popover.Portal>
    <Popover.Content>
      <p>Controlled popover.</p>
      <Popover.Close>Close</Popover.Close>
    </Popover.Content>
  </Popover.Portal>
</Popover.Root>
```

### Styling with data-state

```css
[data-state="open"] {
  opacity: 1;
  visibility: visible;
}

[data-state="closed"] {
  opacity: 0;
  visibility: hidden;
  pointer-events: none;
}
```

## Edge Cases

- **forceMount:** When true, Content stays in DOM when closed. Use `aria-hidden` and `hidden` for accessibility.
- **modal:** Set to true for focus trap; default is false for non-blocking overlays.
- **Positioning:** This library is headless. Use CSS (absolute/fixed), Popper, or Floating UI for positioning.
- **Outside click:** Content closes on outside pointer down by default; Escape also closes.
