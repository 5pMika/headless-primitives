# Tabs

## Overview

Tabs is a headless compound primitive for tabbed interfaces. It matches Radix UI's API: Root, List, Trigger, and Content. Supports roving tabindex, arrow key navigation, and activation modes (automatic vs manual).

**Anatomy:** Root → List → Trigger(s), and Root → Content (per value).

## Accessibility

### Keyboard

- **Arrow Right** / **Arrow Down** — Focus next tab (horizontal/vertical)
- **Arrow Left** / **Arrow Up** — Focus previous tab
- **Home** — Focus first tab
- **End** — Focus last tab
- **Enter** / **Space** — Select tab (when activationMode="manual")
- **Tab** / **Shift+Tab** — Move focus in/out of tablist (roving tabindex)

### Focus

- Roving tabindex: only the focused trigger has `tabIndex={0}`; others have `tabIndex={-1}`
- activationMode="automatic": selecting happens on focus
- activationMode="manual": selecting happens on Enter/Space

### Screen Reader

- List: `role="tablist"`; Triggers: `role="tab"`; Content: `role="tabpanel"`
- `aria-selected`, `aria-controls`, `aria-labelledby` for linking

### Consumer Responsibilities

You must provide:

- Visible focus indicators
- Minimum target size (44x44px recommended)
- Color contrast

## Edge Cases

- **forceMount:** When true, Content stays in DOM when inactive. Use `aria-hidden` and `hidden` for accessibility.
- **activationMode:** "automatic" selects on focus; "manual" requires Enter/Space to select.
- **orientation:** Use "vertical" for side tabs; arrow keys switch to Up/Down.
- **dir="rtl":** Full RTL support; arrow keys respect direction.
- **Disabled trigger:** Not focusable; skipped in keyboard navigation.
