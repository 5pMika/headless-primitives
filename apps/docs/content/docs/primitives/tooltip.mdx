# Tooltip

## Overview

Tooltip is a headless compound primitive for hover/focus-triggered hints. It matches Radix UI's API: Provider, Root, Trigger, and Content. Supports configurable delay, skip delay when moving between tooltips, and Escape to close.

**Anatomy:** Provider (optional) → Root → Trigger, Content. Provider coordinates delay/skip across multiple tooltips.

## Accessibility

### Keyboard

- **Escape** — Closes the tooltip
- **Tab** / **Shift+Tab** — Move focus; tooltip shows on focus, hides on blur
- **Enter** / **Space** — N/A (tooltip is not activated by these on trigger)

### Focus

- Tooltip shows when trigger receives focus
- Tooltip hides when trigger loses focus
- Focus does not move into tooltip content (tooltips are non-interactive)

### Screen Reader

- Content: `role="tooltip"`
- Trigger: `aria-describedby` points to content id
- Accessible name/description comes from tooltip content
- Ensure tooltip content is concise and descriptive

### Consumer Responsibilities

You must provide:

- Visible tooltip styling (background, padding, border, etc.)
- Positioning (CSS, Popper, Floating UI)
- `prefers-reduced-motion` handling if desired (disable or shorten delay)

### Without Provider

Provider is optional. Without it, default delay (700ms) and skip delay (300ms) are used:

```tsx
<Tooltip.Root>
  <Tooltip.Trigger>Hover me</Tooltip.Trigger>
  <Tooltip.Content>Tooltip text</Tooltip.Content>
</Tooltip.Root>
```

### Custom Delay

```tsx
<Tooltip.Root delayDuration={1000}>
  <Tooltip.Trigger>Hover me</Tooltip.Trigger>
  <Tooltip.Content>Shows after 1 second</Tooltip.Content>
</Tooltip.Root>
```

### Controlled

```tsx
const [open, setOpen] = useState(false);

<Tooltip.Root open={open} onOpenChange={setOpen}>
  <Tooltip.Trigger>Click to toggle</Tooltip.Trigger>
  <Tooltip.Content>Controlled tooltip</Tooltip.Content>
</Tooltip.Root>
```

### Styling with data-state

```css
[data-state="open"] {
  opacity: 1;
  visibility: visible;
}

[data-state="closed"] {
  opacity: 0;
  visibility: hidden;
  pointer-events: none;
}
```

## Edge Cases

- **forceMount:** When true, Content stays in DOM when closed. Use `aria-hidden` and `hidden` for accessibility.
- **Provider:** Use when you have multiple tooltips and want skip-delay behavior (moving from one trigger to another shows the next tooltip quickly).
- **Positioning:** This library is headless. Use CSS (absolute/fixed), Popper, or Floating UI for positioning.
- **disableHoverableContent:** Radix supports this to prevent tooltip from staying open when hovering over content. This headless implementation does not include it; add custom logic if needed.
- **Mobile:** Tooltips triggered by focus work on mobile; hover-only tooltips may not. Consider touch-friendly alternatives for critical info.
