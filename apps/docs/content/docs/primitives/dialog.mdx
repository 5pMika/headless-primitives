# Dialog

## Overview

Dialog is a headless compound primitive for modal dialogs. It matches Radix UI's API: Root, Trigger, Portal, Overlay, Content, Title, Description, and Close. Supports focus trap, portal rendering, and full WAI-ARIA patterns.

**Anatomy:** Root → Trigger, Portal → Overlay + Content (Title, Description, Close). The overlay and content are rendered in a portal by default.

## Accessibility

### Keyboard

- **Escape** — Closes the dialog
- **Tab** / **Shift+Tab** — Trapped within dialog when modal
- **Enter** / **Space** — Activate buttons (Trigger, Close)

### Focus

- Focus moves to first focusable element when dialog opens
- Focus is trapped inside the dialog when modal
- Focus returns to trigger when dialog closes
- Portal renders content outside DOM hierarchy for proper stacking

### Screen Reader

- Content: `role="dialog"`, `aria-modal`, `aria-labelledby`, `aria-describedby`
- Title and Description provide accessible name and description
- `data-state` for open/closed

### Consumer Responsibilities

You must provide:

- Visible focus indicators
- Overlay styling (e.g. backdrop)
- Content positioning (this library is headless; no layout/positioning)

## Edge Cases

- **forceMount:** When true, Overlay and Content stay in DOM when closed. Use `aria-hidden` and `hidden` for accessibility; useful for CSS/JS animations.
- **modal:** When false, focus is not trapped; user can interact outside. Use for non-blocking overlays.
- **Portal:** Content renders in portal by default. Ensure overlay and content are positioned correctly (e.g. fixed, full viewport).
- **Nested dialogs:** Focus restoration order is handled; nested dialogs close in reverse order.
