# Primitives

Primitives will be documented here as they are implemented.

Each primitive page follows the structure:

1. **Overview** — What it does
2. **API** — Props (type-derived)
3. **Accessibility** — Keyboard, focus, screen reader
4. **Examples** — Basic, controlled, composition
5. **Edge cases** — Gotchas and limits
