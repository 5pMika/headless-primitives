# Avatar

## Overview

Avatar is a headless compound primitive for user avatars. It matches Radix UI's API: Root, Image, and Fallback. Supports image loading states and fallback when the image fails or is loading.

**Anatomy:** Root → Image (optional), Fallback. Root has `role="img"`; Image and Fallback are mutually exclusive in visibility (image shows when loaded, fallback when not).

## Accessibility

### Keyboard

- Avatar is not interactive by default; no keyboard interaction.
- If used as a link or button, ensure it is focusable and has appropriate role.

### Focus

- Avatar root is not focusable unless it wraps an interactive element (e.g. `asChild` with a button).

### Screen Reader

- Root: `role="img"`
- **Required:** Provide `aria-label` on Root for accessible name (e.g. "User avatar" or "John Doe").
- Image: Use `alt` for decorative images (empty string) or descriptive alt for meaningful images.
- Fallback: Typically initials or icon; ensure Root has `aria-label` since fallback may not be descriptive alone.

### Consumer Responsibilities

You must provide:

- `aria-label` on Root (e.g. "User avatar" or the user's name)
- Image `alt` attribute (empty for decorative: `alt=""`)
- Sizing and styling (this library is headless)

```tsx
<Avatar.Root aria-label="User avatar">
  <Avatar.Fallback>AB</Avatar.Fallback>
</Avatar.Root>
```

### With Delay (Avoid Flash)

When images load quickly, delay showing fallback to avoid a brief flash:

```tsx
<Avatar.Root aria-label="User avatar">
  <Avatar.Image src="/user.jpg" alt="" />
  <Avatar.Fallback delayMs={600}>JD</Avatar.Fallback>
</Avatar.Root>
```

### Loading Status

```tsx
<Avatar.Root aria-label="User avatar">
  <Avatar.Image
    src="/user.jpg"
    alt=""
    onLoadingStatusChange={(status) => console.log(status)}
  />
  <Avatar.Fallback>JD</Avatar.Fallback>
</Avatar.Root>
```

### Styling with data-state

```css
[data-state="loaded"] {

[data-state="loading"],
[data-state="error"] {
```

## Edge Cases

- **aria-label:** Always provide `aria-label` on Root. Screen readers need an accessible name for `role="img"`.
- **delayMs:** Use when images typically load fast to avoid fallback flash. For slow networks, fallback appears after delay.
- **Image error:** When image fails to load, fallback is shown. Ensure fallback has meaningful content (e.g. initials).
- **No Image:** When Image is omitted, Fallback is always shown.
- **asChild:** When true on Root, merges props onto single child. Child must accept ref and span props. Ensure `role="img"` and `aria-label` are preserved.
