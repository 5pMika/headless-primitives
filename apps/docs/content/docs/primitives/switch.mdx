# Switch

## Overview

Switch is a headless single-component primitive for on/off toggles. It matches Radix UI's API with `checked`, `defaultChecked`, `onCheckedChange`, and supports form integration via `name` and `value`.

**Anatomy:** Single component (no compound parts). Renders a button with `role="switch"` and `aria-checked`.

## Accessibility

### Keyboard

- **Enter** / **Space** — Toggle the switch
- **Tab** / **Shift+Tab** — Move focus in/out

### Focus

- Switch is focusable when not disabled
- Native button behavior

### Screen Reader

- `role="switch"`
- `aria-checked` (true/false)
- `aria-disabled` when disabled
- Accessible name from visible text, `aria-label`, or `aria-labelledby`

### Consumer Responsibilities

You must provide:

- Visible focus indicators
- Visual representation of checked/unchecked (e.g. track + thumb)
- Minimum target size (44x44px recommended)

## Edge Cases

- **asChild:** When true, merges props onto single child. Child must accept ref and button props.
- **Form submission:** When `name` is set, the switch submits its `value` when checked (or nothing when unchecked). Use a hidden input if you need to always submit a value.
- **required:** `aria-required` is set; form validation behavior depends on your form library.
