# Checkbox

## Overview

Checkbox is a headless compound primitive for checkboxes. It matches Radix UI's API: Root and Indicator. Supports tri-state (`checked`, `unchecked`, `indeterminate`) and includes a hidden native input for form submission.

**Anatomy:** Root (button wrapper + hidden input) → optional Indicator (visual). The Root handles semantics; Indicator is for custom visuals.

## Accessibility

### Keyboard

- **Enter** / **Space** — Toggle (cycles: unchecked → checked → unchecked; or indeterminate → checked → unchecked)
- **Tab** / **Shift+Tab** — Move focus in/out

### Focus

- Checkbox is focusable when not disabled
- Hidden native input preserves form semantics

### Screen Reader

- Root: `role="checkbox"`, `aria-checked` (true/false/"mixed" for indeterminate)
- `aria-disabled` when disabled
- Accessible name from visible text, `aria-label`, or `aria-labelledby`

### Consumer Responsibilities

You must provide:

- Visible focus indicators
- Visual representation of checked/unchecked/indeterminate
- Minimum target size (44x44px recommended)

### Tri-state (Indeterminate)

```tsx
const [checked, setChecked] = useState<'indeterminate' | boolean>('indeterminate');

<Checkbox.Root
  checked={checked}
  onCheckedChange={setChecked}
  aria-label="Select all"
>
  <Checkbox.Indicator>{checked === 'indeterminate' ? '−' : checked ? '✓' : ''}</Checkbox.Indicator>
</Checkbox.Root>
```

### Controlled

```tsx
const [checked, setChecked] = useState(false);

<Checkbox.Root checked={checked} onCheckedChange={setChecked} aria-label="Subscribe">
  <Checkbox.Indicator>✓</Checkbox.Indicator>
</Checkbox.Root>
```

### Form Integration

```tsx
<form onSubmit={(e) => { e.preventDefault(); const fd = new FormData(e.currentTarget); console.log(fd.get('agree')); }}>
  <Checkbox.Root name="agree" value="yes" aria-label="I agree">
    <Checkbox.Indicator>✓</Checkbox.Indicator>
  </Checkbox.Root>
  <button type="submit">Submit</button>
</form>
```

### Styling with data-state

```css
[data-state="checked"] .indicator {
  display: block;
}

[data-state="unchecked"] .indicator {
  display: none;
}

[data-state="indeterminate"] .indicator {
  display: block;
  /* e.g. minus icon */
}

[data-disabled] {
  opacity: 0.5;
  cursor: not-allowed;
}
```

## Edge Cases

<Callout type="info">
  **Tri-state cycle:** When indeterminate, the first activation goes to checked, then unchecked. The exact cycle depends on your `onCheckedChange` logic.
</Callout>

- **forceMount:** Not applicable; Checkbox has no overlay.
- **Hidden input:** Root includes a hidden `input[type="checkbox"]` for form submission. The `indeterminate` state is set on this input for proper semantics.
- **asChild:** When true on Root, merges props onto single child. Child must accept ref and button props.
- **Indicator:** Optional. Use for custom checkmark/minus icons. Root works without it if you style the button directly.
