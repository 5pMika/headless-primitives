# Toast

## Overview

Toast is a headless multi-part primitive for temporary notification messages. It matches Radix UI's API: Provider, Viewport, Root, Title, Description, Action, and Close. Supports auto-dismiss, pause on hover/focus/blur, swipe-to-close, and F8 hotkey to focus the viewport.

**Anatomy:** Provider → Viewport + Root(s). Each Root contains optional Title, Description, Action, and Close. Toasts are portaled into the Viewport.

## Accessibility

### Keyboard

- **F8** — Focus toast viewport (default hotkey)
- **Tab** / **Shift+Tab** — Move focus between focusable elements
- **Space** / **Enter** — Activate Action or Close (closes toast)
- **Escape** — Close the focused toast

### Focus

- Viewport is focusable via hotkey (`tabIndex={-1}`)
- No focus trap; toasts are non-modal
- Action and Close buttons close the toast on activation

### Screen Reader

- Root: `role="status"`, `aria-live` (assertive for foreground, polite for background), `aria-atomic="true"`
- Action: `altText` is required — provide alternative instruction (e.g. "Undo via Alt+U")
- Close: Consumer must provide accessible label (e.g. "Dismiss")

### Consumer Responsibilities

You must provide:

- Visible focus indicators
- Minimum target size (44x44px recommended)
- Color contrast
- Hotkey discoverability (document F8 or custom hotkey)

### Duplicate Toasts (State Pattern)

When a toast must appear every time the user clicks, use state to render multiple instances:

```tsx
const [count, setCount] = useState(0);

<Toast.Provider>
  <Toast.Viewport />
  <button onClick={() => setCount((c) => c + 1)}>Save</button>
  {Array.from({ length: count }).map((_, i) => (
    <Toast.Root key={i}>
      <Toast.Title>Saved!</Toast.Title>
    </Toast.Root>
  ))}
</Toast.Provider>
```

## Edge Cases

- **forceMount:** When true, the toast stays in the DOM when closed. Use for exit animations. Apply `aria-hidden` and `hidden` when closed.
- **foreground vs background:** Use `type="foreground"` for user-action toasts (announced immediately). Use `type="background"` for background task toasts (announced when screen reader is idle).
- **altText on Action:** Required. Provide an alternative way to act (e.g. keyboard shortcut or permanent UI location).
- **Viewport order:** Viewport must render before Root(s) so the portal target exists. Typical structure: Provider → Viewport → Root(s).
- **Portal:** Toasts are portaled into the Viewport. If Viewport is not yet mounted, toast renders inline until the next render.
