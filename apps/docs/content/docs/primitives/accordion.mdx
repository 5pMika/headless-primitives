# Accordion

## Overview

Accordion is a headless multi-part primitive for expandable/collapsible content panels. It matches Radix UI's API: Root, Item, Header, Trigger, and Content. Supports single or multiple open items, collapsible mode, and full WAI-ARIA keyboard patterns.

**Anatomy:** Root → Item(s) → Header → Trigger, and Item → Content. Each Item groups a Trigger (header) with its Content (panel).

## Accessibility

### Keyboard

- **Enter** / **Space** — Toggle the focused item (native button activation)
- **Arrow Up/Down** — Move focus between triggers (vertical orientation)
- **Arrow Left/Right** — Move focus between triggers (horizontal orientation)
- **Home** — Focus first trigger
- **End** — Focus last trigger
- **Tab** / **Shift+Tab** — Move focus in/out of accordion (roving tabindex: only one trigger has tabindex="0")

### Focus

- Roving tabindex: only the focused trigger has `tabIndex={0}`; others have `tabIndex={-1}`
- Disabled items are skipped and not focusable
- Focus order follows DOM order

### Screen Reader

- Trigger: `aria-expanded`, `aria-controls` (content id)
- Content: `role="region"`, `aria-labelledby` (trigger id), `aria-hidden` when closed
- Accessible name from visible text, `aria-label`, or `aria-labelledby`

### Consumer Responsibilities

You must provide:

- Visible focus indicators
- Minimum target size (44x44px recommended)
- Color contrast

### Collapsible (Single)

When `collapsible` is true, clicking an open item closes it.

```tsx
<Accordion.Root type="single" collapsible defaultValue="item-1">
  <Accordion.Item value="item-1">
    <Accordion.Header>
      <Accordion.Trigger>Section 1</Accordion.Trigger>
    </Accordion.Header>
    <Accordion.Content>Content 1</Accordion.Content>
  </Accordion.Item>
</Accordion.Root>
```

### Multiple

Multiple items can be open at once.

```tsx
<Accordion.Root type="multiple" defaultValue={['item-1', 'item-3']}>
  <Accordion.Item value="item-1">
    <Accordion.Header>
      <Accordion.Trigger>Section 1</Accordion.Trigger>
    </Accordion.Header>
    <Accordion.Content>Content 1</Accordion.Content>
  </Accordion.Item>
  <Accordion.Item value="item-2">
    <Accordion.Header>
      <Accordion.Trigger>Section 2</Accordion.Trigger>
    </Accordion.Header>
    <Accordion.Content>Content 2</Accordion.Content>
  </Accordion.Item>
</Accordion.Root>
```

### Controlled

Control the open state from outside.

```tsx
const [value, setValue] = useState('item-1');

<Accordion.Root type="single" collapsible value={value} onValueChange={setValue}>
  <Accordion.Item value="item-1">
    <Accordion.Header>
      <Accordion.Trigger>Section 1</Accordion.Trigger>
    </Accordion.Header>
    <Accordion.Content>Content 1</Accordion.Content>
  </Accordion.Item>
</Accordion.Root>;
```

### Styling with data-state

Use `data-state` for styling open/closed and disabled states:

```css
[data-accordion-trigger][data-state='open'] {
  /* expanded trigger styles */
}

[data-accordion-trigger][data-state='closed'] {
  /* collapsed trigger styles */
}

[data-disabled] {
  opacity: 0.5;
  cursor: not-allowed;
}
```

## Edge Cases

- **forceMount:** When `forceMount` is true on Content, the panel stays in the DOM when closed. Use `aria-hidden` and `hidden` for accessibility; useful for CSS/JS animations.
- **orientation:** Use `orientation="horizontal"` for left/right layouts; arrow keys switch to Left/Right.
- **dir="rtl":** Full RTL support; keyboard navigation respects direction.
- **collapsible in single mode:** Without `collapsible`, clicking an open item does nothing. With `collapsible`, it closes.
- **Disabled item:** Disabled items are not focusable and do not respond to clicks.
