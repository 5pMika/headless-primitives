# Form

## Overview

Form is a headless primitive for building accessible forms with validation. It mirrors the Radix UI Form API and is built on the native HTML5 Constraint Validation API. Supports built-in validation (`required`, `min`, `max`, `pattern`), custom validation via `setCustomValidity`, and server-side validation with `serverInvalid` and `onClearServerErrors`.

**Anatomy:** Root → Field(s) → Label, Control, Message, ValidityState. Each Field groups a Label, Control, and optional Message. Form.Submit renders a submit button. Form.ValidityState exposes validity via render prop.

## Accessibility

### Keyboard

- **Tab** / **Shift+Tab** — Move focus between form controls (native behavior)
- **Enter** — Submits form when focus is on submit button (native behavior)

### Focus

- Focus moves to first invalid control on submit when validation fails
- Native tab order follows DOM order
- Disabled controls are not focusable

### Screen Reader

- Label associated with control via `htmlFor` / `id`
- Control has `aria-describedby` pointing to message element
- Control has `aria-invalid` when invalid
- Message has `role="alert"` and `aria-live="polite"` for announcements
- Keep validation messages short (max 10 words recommended)

### Consumer Responsibilities

You must provide:

- Visible focus indicators on all form controls
- Minimum target size (44x44px recommended for touch)
- Color contrast for labels and messages
- Full page testing with real screen readers

### With Validation

```tsx
<Form.Root onSubmit={(e) => { e.preventDefault(); console.log(new FormData(e.currentTarget)); }}>
  <Form.Field name="email">
    <Form.Label>Email</Form.Label>
    <Form.Control type="email" required />
    <Form.Message match="valueMissing">Required</Form.Message>
    <Form.Message match="typeMismatch">Enter a valid email</Form.Message>
  </Form.Field>
  <Form.Field name="age">
    <Form.Label>Age</Form.Label>
    <Form.Control type="number" min={18} max={120} required />
    <Form.Message match="rangeUnderflow">Must be 18 or older</Form.Message>
    <Form.Message match="rangeOverflow">Invalid age</Form.Message>
  </Form.Field>
  <Form.Submit>Submit</Form.Submit>
</Form.Root>
```

### Server Errors

```tsx
const [serverError, setServerError] = useState<string | null>(null);

<Form.Root
  onClearServerErrors={() => setServerError(null)}
  onSubmit={async (e) => {
    e.preventDefault();
    const res = await fetch('/api/login', { method: 'POST', body: new FormData(e.currentTarget) });
    if (!res.ok) setServerError('Invalid credentials');
  }}
>
  <Form.Field name="email" serverInvalid={!!serverError}>
    <Form.Label>Email</Form.Label>
    <Form.Control type="email" required />
    <Form.Message forceMatch>{serverError ?? 'Email is required'}</Form.Message>
  </Form.Field>
  <Form.Submit>Submit</Form.Submit>
</Form.Root>
```

### Form.Control with textarea (asChild)

```tsx
<Form.Field name="bio">
  <Form.Label>Bio</Form.Label>
  <Form.Control asChild>
    <textarea rows={4} placeholder="Tell us about yourself" />
  </Form.Control>
  <Form.Message match="tooLong">Maximum 500 characters</Form.Message>
</Form.Field>
```

## Edge Cases

- **Form.Control with textarea/select:** Use `asChild` to wrap `<textarea>` or `<select>`. The control receives `id`, `name`, `aria-describedby`, `aria-invalid` from the field.
- **Custom validation:** Use `element.setCustomValidity(message)` in `onInput` or `onBlur`. Clear with `setCustomValidity("")` when valid.
- **Form.Message outside Field:** Pass the `name` prop to associate with a field. The message must be within Form.Root.
- **RTL:** Form respects `dir="rtl"` on the form or ancestor.
- **prefers-reduced-motion:** No animations are applied by the primitive; consumer controls any motion.
